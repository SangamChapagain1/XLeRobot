XLeRobot Bimanual WiFi Teleop

How two SO101 leader arms on a MacBook control two follower arms on a Jetson AGX Orin over WiFi

Flowchart 1

Original XLeRobot vs Our Fork

What the system looked like originally and what we changed to enable remote bimanual WiFi teleop with dataset recording.

Original XLeRobot
All 17 motors on one machine (local USB only)
Leader input: VR headset / keyboard / gamepad
3 cameras: front + 2 wrist (all local)
Head motors (2) + base wheels (3) always required
ZMQ host/client existed but only for base robot
No dataset recording during teleop
Relative imports: from ..robot import Robot
Our Fork (Working Now)
Split across WiFi: leaders on Mac, followers on Jetson
Leader input: 2x physical SO101 arms via USB serial
1 camera: front only (single-camera mode)
Head/base motors optional (graceful skip if absent)
ZMQ extended for bimanual 12-DOF + camera streaming
LeRobot dataset recording built into teleop client
Absolute imports: from lerobot.robots.robot

Flowchart 3

Single Control Loop Iteration (30Hz)

What happens every 33ms in one cycle of the teleop loop — from reading leader arms to displaying the camera feed and recording data.

1
MacBook

Read Leader Arms

read_leaders() calls SO101Leader.get_action() on both USB serial buses. Returns 12 raw joint positions (6 left + 6 right).

2
MacBook

Smooth + Deadband Filter

smooth_action() applies EMA smoothing (α=0.35) and deadband filtering (0.25 joints, 0.4 grippers) to eliminate servo jitter.

3
WiFi → ZMQ

Send Action Over ZMQ

robot.send_action() serializes 17 values as JSON (12 arm + 2 head + 3 base, head/base zeroed) and pushes to Jetson via ZMQ PUSH :5555. ~500 bytes per message.

4
Jetson

Drive Follower Arms

xlerobot_host.py receives the JSON, calls robot.send_action() which writes Goal_Position to left arm (Bus1) and right arm (Bus2) via Feetech serial protocol.

5
Jetson

Read Observation + Camera

robot.get_observation() reads Present_Position from both follower arms (12 joints), captures front camera, JPEG-encodes (quality 90), base64-encodes, bundles into JSON.

6
WiFi ← ZMQ

Stream Observation Back

Jetson pushes the observation JSON (~30-50KB with JPEG) via ZMQ PUSH :5556. Mac receives with CONFLATE=1 — always gets latest frame, drops old ones.

7
MacBook

Display Camera + Check Keys

Decodes base64 JPEG, displays in OpenCV window with REC/VLA overlay. Checks keyboard: R toggles recording, V toggles VLA mode, Q quits.

8
MacBook (background thread)

Record & Save Episode

If recording: builds LeRobot frame with action (12-D leader), observation.state (12-D follower), observation.images.front (camera). After 300 frames, dataset.save_episode() writes to disk. Optionally pushes to HuggingFace Hub.
